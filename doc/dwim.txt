*dwim.txt*  DWIM-open files from filename:line:col strings

INTRODUCTION                                    *dwim*

A tiny Vim plugin that opens files from a filename:line:col string and
jumps to the requested location. Useful for pasting paths produced by
compilers, linters, terminal output, or git diffs.

COMMANDS                                        *dwim-commands*

                                                *:DWIM*
:[count]DWIM [++curwin] [{target}]
                        Open {target} and jump to the location encoded
                        in it. {target} may be:

                          filename
                          filename:line
                          filename:line:col

                        When called with no argument, {target} is read
                        from the OS clipboard (the |quote+| register).
>
                          :DWIM src/foo.ts:42:7
                          :DWIM b/src/foo.ts:42
                          :DWIM
<
                        Window placement is controlled by the usual
                        |:command-modifiers|:

                          :DWIM {target}            edit in the current
                                                    window (default)
                          :0DWIM {target}           force current window
                                                    (shorthand for
                                                    ++curwin)
                          :DWIM ++curwin {target}   force the current
                                                    window, ignoring any
                                                    modifiers
                          :vert DWIM {target}       vertical split
                          :aboveleft DWIM {target}  horizontal split
                                                    above
                          :belowright DWIM {target} horizontal split
                                                    below
                          :tab DWIM {target}        open in a new tab

                        Internally, :DWIM switches from |:edit| to
                        |:split| whenever a window/tab modifier is
                        present, so |:vertical|, |:aboveleft|,
                        |:leftabove|, |:belowright|, |:rightbelow|,
                        |:topleft|, |:botright|, and |:tab| all Just
                        Work.

                        A trailing |!| is passed through to the
                        underlying |:edit| / |:split| so unsaved changes
                        in the current buffer can be discarded.

NORMALIZATION                                   *dwim-normalization*

Before parsing, the input is cleaned up:

  - Newlines, carriage returns, NUL bytes, and literal ^@ sequences
    introduced by terminal copy/paste are stripped.
  - Leading and trailing whitespace is trimmed.
  - Leading path components are peeled off: the path is tried as-is,
    then with leading components stripped one at a time, and the first
    variant that exists on disk is opened. For "/app/src/foo/bar.ts"
    this tries, in order, "/app/src/foo/bar.ts", "app/src/foo/bar.ts",
    "src/foo/bar.ts", "foo/bar.ts", "bar.ts". When nothing matches the
    original path is kept, so opening a brand-new file still works.
    This also removes git diff prefixes "a/" and "b/".
  - TypeScript / MSBuild locations of the form file(line,col) are
    rewritten to file:line:col. Any trailing diagnostic text after
    the location is dropped, so this works:
>
      :DWIM foo.ts(280,13): error TS2339: Property 'x' does not exist
<
    and opens foo.ts at line 280, column 13.

MAPPINGS                                        *dwim-mappings*

No mappings are created by default. A convenient one:
>
  nnoremap <leader>o :DWIM<CR>
<
